# Portfolio 2.0 project

## Idea

Make a photography portfolio engine with the main characteristic of:
Working as static html system on serwer
Having local engine that allow all the dynamic edit of portfolio. 

### Workflow

1. Run local engine - edit portfolio, add/remove pictures, galleries etc.
2. Build the release state 
3. Upload the release state to server (github io) 
4. the site works just like static webpage 

### Features requirements 
- The portfolio to by with the possibility to see big photo after click (i want to shot my photos to people reasonable hires)
- Showing the thumbnails 
- Nice clean light themed design. 
- Can have like master gallery with sub galleries 
- Gallery can have a title and description text, 
- Photo might have a title and description, but it's not required. 
- Can set photos order manually but def is newest add on top 
- can share a photo or gallery via link (if this is possible to do on static)
- have a nice landing page, with text and my photo, and a section with latest changes
- option to add page - article like - a blog like - text with pics in in it.
